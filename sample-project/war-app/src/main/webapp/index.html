Product Management WAR Application

Traditional servlet-based web application for demonstrating call graph analysis

Call Graph Analysis Demo

This WAR application demonstrates a deep call hierarchy suitable for call graph analysis:

Call Hierarchy Levels (9 levels deep):

  1. Tomcat Container - Servlet container lifecycle
  2. ProductServlet - HTTP request handling (doGet, doPost, etc.)
  3. Servlet Methods - Request processing methods
  4. ProductService - Business logic layer
  5. Service Support - Validation, caching services
  6. ProductDao - Data access operations
  7. DAO Operations - Database query execution
  8. DatabaseUtil - Database utility operations
  9. Low-level Operations - Connection setup, logging

Available REST API Endpoints:

  • GET /api/products - Get all products
  • GET /api/products/{id} - Get product by ID
  • POST /api/products - Create new product
  • PUT /api/products/{id} - Update product
  • DELETE /api/products/{id} - Delete product

Sample Product JSON:

{
  "name": "Sample Product",
  "description": "Product description",
  "price": 29.99,
  "quantity": 100,
  "category": "Electronics"
}

Usage for Call Graph Analysis:

To analyze this application with the call graph tool:

  1. Build the WAR: ./gradlew build
  2. Analyze with tool: ./gradlew bootRun --args="--verbose --include=com.example.warapp build/libs/war-app-1.0.0.war"
  3. Observe the multi-level call hierarchy in the output

Note: This application uses manual dependency injection instead of frameworks like Spring to ensure clear visibility of method calls for static analysis.
